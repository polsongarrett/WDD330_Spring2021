JavaScript: Novice to Ninja, 2nd Edition

Chapter 9

Linting tools can be found on the web for testing your code for errors and sloppy coding practices, although I find many of these tools to be too picky and opinionated.

Two tools for feature detection include Modernizr, a library, and caniuse.com, which tells you which features are currently supported in different browsers.

I use console.log frequently for debugging, but there's also console.trace, which outputs the stack trace to the console.

JavaScript has some built-in Error objects that the browser will throw when different kinds of exceptions occur.

I've always been taught that Test-driven development (TDD) is considered best practice, but sometimes it can get in the way if we always put it first, no matter what.

There are many testing frameworks for JavaScript, but this author focuses on using Jest. Though they don't go into it, it seems that you first need to install Node.js and npm, then you can install Jest. After installing Jest, here's an example from the book showing how to test some code with Jest: 


  function (number) {
    if (number < 0) {
        throw new RangeError("You can't find the square root of negative numbers")  
    }
    return Math.sqrt(number);
  };

  test('square root of 4 is 2', () => {
    expect(squareRoot(4)).toBe(2);
  });
