A variable of the Symbol type is unique. If you create two symbols with the same description they will have different values. It seems this porperty is useful for avoiding clashes with other developers' code and giving an object more than one property with the same name.

Symbol("id") == Symbol("id") // false

const person = {
firstName: "John",
};

let id = Symbol('id');
person[id] = 140353;
// person[id] = 140353
// person.id is still undefined

The in operator is useful for checking if an object has a certain porperty. The hasOwnProperty() method, which all JavaScript objects inherit, accomplishes the same thing. Specifically, it determins if a property that is not inherited belongs to an object.

'city' in person;
<< false

person.hasOwnProperty('city');
<< false

Object.keys() is useful for returning an array of all the properties in an object that are not inherited. You could then iterate through that array with a for-of loop.

Object.values() does the same thing except it returns an array of the values instead of the properties.

Object.entries() returns an array of key-value pairs, so it's like a combination of the previous two methods.

You can create functions with the same name by putting them in their own namespace. 

     
  const nameSpace = {        
    foo() {                  
      // code here           
    }                        
  };                         
                              
  const anotherNameSpace = { 
    foo() {                  
      // code here           
    }                        
  };                         
                              
  namespace.foo();           
  anotherNameSpace.foo();
