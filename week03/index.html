Chapter 5

A variable of the Symbol type is unique. If you create two symbols with the same description they will have different values. It seems this porperty is useful for avoiding clashes with other developers' code and giving an object more than one property with the same name. 


  Symbol("id") == Symbol("id") // false 
  const person = { 
    firstName: "John", 
  }; 

  let id = Symbol('id'); 
  person[id] = 140353; 
  // person[id] = 140353 
  // person.id is still undefined

The in operator is useful for checking if an object has a certain porperty. The hasOwnProperty() method, which all JavaScript objects inherit, accomplishes the same thing. Specifically, it determins if a property that is not inherited belongs to an object. 


  'city' in person; 
  << false      

  person.hasOwnProperty('city');
  << false

Object.keys() is useful for returning an array of all the properties in an object that are not inherited. You could then iterate through that array with a for-of loop.

Object.values() does the same thing except it returns an array of the values instead of the properties.

Object.entries() returns an array of key-value pairs, so it's like a combination of the previous two methods.

You can create functions with the same name by putting them in their own namespace. 

     
  const nameSpace = {        
    foo() {                  
      // code here           
    }                        
  };                         
                              
  const anotherNameSpace = { 
    foo() {                  
      // code here           
    }                        
  };                         
                              
  namespace.foo();           
  anotherNameSpace.foo();

Chapter 6

This chapter deals with DOM (Document Object Model), which I had never payed much attention to before. It is extremely useful for manipulating HTML nodes!

  • Every element of an HTML page is a node in a tree, which means everything has a parent and a child except for the last child and the first parent.
  • The easiest way to access a node's attributes is to assign the node reference to a variable and use dot notation to access its attributes.
  • Use queryselector() for getting elements by id, class, tag, pseudo selectors, and more!
  • You can delete and create nodes through DOM (createElement(), createTextNode(), appendChild(), removeChild() etc.)
  • CSS can also be changed through the DOM.
  • "The node lists returned by the document.getElementsByClassName() and document.getElementsByTagName() methods are live collections that will update to reflect any changes on the page." This means they update automatically without you having to make function calls. This can be expensive performance-wise.

Chapter 7

This chapter was about event handlers. Again, I thought I knew enough about event handlers, but I learned some extra useful things here.

  • Don't use inline-event handlers in the HTML if you don't have to. Keep all your JavaScript in one place.
  • When you add an event listener, you can access the event object by passing it as a parameter to the function that gets called. The event object has some useful properties, such as "type","coordinates", and "target." Target is useful for identifying the node that fired an event.
  • Event Propagation is when an event fires on a triggering element as well as either all of its parents (bubbling) or all of its children (capturing) . This could be useful if you want other events to fire when either a child or parent node's event is triggered. You can stop event propagation from happening by calling stopPropagation() inside the triggered function.
  • "Event delegation can be used to attach an event listener to a parent element in order to capture events that are triggered by its child elements." For example, if you wanted to fire an event to highlight all <li>s any time a <li> was clicked, instead of adding an event listener to every <li> or even selecting those <li>s by class, assign an event listener to the <ul>. Use the target property to dynamically identify the nodes the event fires on. Because of event propagation, the event will fire on every child node of <ul>. 
     
  ulElement.addEventListener('click', event =>
    event.target.classlist.toggle('highlight'));
